javascript--核心点的理解

这是最近读细读高程五、六、七章的javascript核心内容的一些总结吧。

javascript–创建函数发生的事

1. 在创建函数的时候，在作用域链这块，会预先创建一个包含全局变量对象的的作用域链，这个作用域链会保存在[[scope]]属性中， 2. 创建完之后，当我们第一次调用函数的时候， 2.1 首先我们会创建一个执行环境以及相应的作用域链；

这个作用域链的创建过程是首先将预先创建作用域链中的[[scope]]属性中的对象复制到我们将要够贱的作用域链，然后将函数的活动对象一个个推入到作用域链的最前端，

2.2 接着，会使用agreemens以及其他的命名参数的值来初始化函数的活动对象；

javascript–仅有的实现继承

作为一门编程语言少不了继承这一说，js也当然也有其独特的继承方式。简单说就是一个原型对象的实例赋值给另一个对象的原型。 这里说下《js高程》中的关于继承的介绍：

  1. 借用原型链继承，继承的主要方法

    function Parent2(){

    this.name = "parent2";
this.play = \[1,2,3\];

    }
    function Child2(){

    this.type = "child2";

    }
    Child2.prototype = new Parent2();

    不足之处: 这样的继承会使原型的属性是共享的。

  2. 借用构造函数继承

    function Parent1(){

    this.name = "parent1"

    }
    function Child1(){

    Parent1.call(this);
this.type = "child1";

    }

    不足之处：Child1无法继承Parent1的原型对象，并没有真正的实现继承（部分继承）

  3. 组合继承

    function Parent3(){

    this.name = "parent3";
this.play = \[1,2,3\];

    }
    function Child3(){

    Parent3.call(this);
this.type = "child3";

    }
    Child3.prototype = Object.create(Parent3.prototype);
    Child3.prototype.constructor = Child3;

javascript–少不了的this问题

js高级程序设计里面对this的介绍是“this对象是在运行时基于函数的执行环境绑定的”，大家都很容易理解成当一个函数被当做一个对象的方法调用时，this就等于这个对象，或是这个对象的一个指针，基本上这样理解是没问题的，不过在匿名函数中，匿名函数的执行环境具有全局性，所以匿名函数的this是指向window(这里说下，js里面的call()和apply()可以改变函数的执行环境，因此this也会指向其他对象)。

javascript–闭包与递归

js中闭包是一个很独特的点，具体说起来也很复杂，高程中的介绍是有权访问顶一个作用域中的变量的函数，其实这里就有两种情况， 处于底层的活动对象是可以动过作用域链方外外层的活动变量，而在我们理解的闭包，在满足这个条件的同时，还存在着外层的作用域还可以访问底层的作用域中的活动变量，这才叫闭包，还有一个很重要的点，就是函数的在使用完后，js会销毁函数中的变量，可是闭包就有一点不一样，第一因为匿名函数的原因，还有放回这个匿名匿名函数使得这个函数得不到及时的销毁，这样闭包中的变量就会一直存在，这样会影响内存的使用。闭包感觉东西还是很多的，先简单说点。 其次递归这里，js高程中推荐用

agreements.call

来调用本身，具体原因，去看书吧。
